Squeezing the most out of AI agents is a game of context curation. Increasingly, context curation revolves around giving the agent the tools it needs to curate its own context. Frontloading a ton of instructions that *might* be needed is ineffective and inefficient when every token in the context window is fighting for attention. The key is progressive disclosure. This is what [skills](https://agentskills.io/home) do. Instead of flooding the agent with instructions, give it a list of skill titles and summaries that are basically saying "Hey, if you need to do X, read me." This is also how we write documentation for humans! We don't make one massive wall of text, we categorize documentation so that humans can look at a list of titles in a sidebar and jump right to the information they actually need. The fact that something this obvious is what's behind the massive hype around skills right now should be an indication of how much low hanging fruit there is. See [The Intent Layer](https://www.intent-systems.com/learn/intent-layer) for a lesser known idea that also uses the principle of progressive disclosure that I think is very good and worth experimenting with.
